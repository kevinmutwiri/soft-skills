## User Journey Mapping

Part of successful solution creation is a tool called a user journey map.

A user journey map is like a roadmap that shows the path a person takes when using a product or service. It helps us understand how users feel and what they do at each step of their experience. By using this map, we can see where users might face challenges or have positive experiences and make improvements accordingly.

When designing the best user experiences, it’s important to understand and visualize the user experience a customer has when they interact with your product or service. This helps to identify pain points, opportunities for improvement, and moments that matter to users.

As you watch [this 3-minute video](https://www.youtube.com/watch?v=mSxpVRo3BLg), you should be able to answer the following questions:

- What is a journey map used for?
- What should the journey map include?
- What are the 9 key steps?


## User Journey Mapping: Step by Step Guide

1. Define the Purpose
Before diving into user journey mapping, first revisit your goal for creating the product or service in the first place. Why did you develop it, who will it serve and how it will offer this service?

2. Gather User Insights
This stage focuses on understanding the users’ needs, motivations, and pain points before they engage with the product or service. User needs and pain points can be included here to capture the initial insights and understanding of what users are looking for and the challenges they face.

3. Generate a List of User Touch Points
Start by listing all the touch points a user interacts with when engaging with your solution (product or service). Touch points include website visits, social media interactions, customer support calls, physical store visits, and more. Capture every step of the user’s journey, from initial awareness to post-purchase interactions. User needs and pinpoint statements can be included at each touchpoint to identify specific user requirements and expectations during those interactions.

4. Create an Empathy Map
Create an empathy map to highlight user actions at each touch point. This will show each user’s experience and how they feel interacting with your product. What are they doing, thinking, or feeling? Add these actions and emotions as data points aligned with the touch points on the timeline. This helps capture the user’s mindset and experiences throughout their journey.

5. Brainstorm and Highlight Challenges and Opportunities
Identify bottlenecks and areas where the user might face difficulties or frustrations while using your solution. Mark them on the user journey map. Also, identify opportunities to improve the user experience or add value at specific touch points. Highlight these as well.

6. Complete an Affinity Diagram
This will help organize user insights, identify pain points, and decision making, and foster collaboration among teams.

7. Create a Visual Representation
Choose a format for your user journey map. It can be a simple linear timeline or a more complex diagram. Draw a horizontal line representing the timeline of the user journey. Mark key touchpoints along this line.

8. Refine and Digitize
Make your visual clear and captivating for the audience.

9. Share Your User Journey Map
Share the user journey map with stakeholders, designers, or other team members. Seek feedback and validation of the map’s accuracy and insights. Iterate and refine the map based on the feedback received.

Finally, continuously update and evolve: Remember that user journey mapping is an ongoing process. As you gather more data and receive user feedback, update and refine the user journey map to reflect changes and new insights. This ensures that your understanding of the user experience remains current.

Some good and bad examples of pain points:

A good example of a pain-point statement: “My user needs a way to stop their knees from bleeding”

A bad example of a pain-point statement: “My user needs a bandaid.”

A good example of a pain-point statement: “My user is frustrated with the lack of public transportation information for their area and needs a reliable solution that provides adequate up to information on matatus, boda bodas and minivan.”

A bad example of a pain point statement: “My user needs transport.”

Other good examples:

User Need Statement: “My user needs a convenient and user-friendly mobile banking app that allows them to easily manage their finances on the go, check account balances, transfer funds, and pay bills.”

User Pain Point Statement: “My user is frustrated with the manual and time-consuming expense tracking process, often leading to errors and difficulty in managing their personal finances efficiently.”
